# WIN-019 公网探测禁用缓存

**基线状态：应通过**

## 测试用途

验证每次 Windows 公网主动探测都读取当前网络响应，不使用旧 HTTP 缓存伪造成功或失败。

## 测试背景

网络刚断开或认证门户刚出现时，缓存的 Microsoft Connect Test 成功正文会造成公网状态滞后。

## 测试输入条件

1. 能控制 Microsoft Connect Test 响应的透明代理或测试网关。
2. 初始返回标准成功响应。
3. 随后切换为失败、重定向或非预期正文。

## 测试场景

1. 成功响应后切换为 TCP 失败。
2. 成功响应后切换为 HTTP 200 非预期正文。
3. 认证门户响应后恢复标准成功正文。

## 执行步骤

1. 让插件完成一次成功探测。
2. 修改测试网关响应并触发网络变化。
3. 观察下一次主动探测结果。
4. 再恢复响应并重复。

## 通过要求

1. 每次结果反映测试网关的当前响应。
2. 不复用旧 200 成功正文。
3. 认证门户和恢复都能在下一次有效探测中体现。
4. `checkedAt` 更新，且 activeProbe 证据与抓包一致。

## 反例与失败判定

1. 网关已改为失败但插件继续返回缓存成功，判定失败。
2. 恢复后仍返回缓存的认证门户，判定失败。
3. 抓包中没有新请求却生成新的成功探测，判定失败。

## 证据留存

1. 保存代理规则切换时间。
2. 保存每次快照和抓包。
3. 记录 HTTP cache 相关请求头和响应头。
