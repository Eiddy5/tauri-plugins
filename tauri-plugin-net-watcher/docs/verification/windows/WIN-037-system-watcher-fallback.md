# WIN-037 系统事件监听失败降级

**基线状态：应通过；需要故障注入构建**

## 测试用途

验证 `NotifyIpInterfaceChange` 初始化失败时，插件仍能启动并通过定时探测更新状态。

## 测试背景

系统事件是低延迟唤醒优化，不应成为插件启动的单点故障。降级后功能可用但变化发现延迟允许增加。

## 测试输入条件

1. 提供测试构建或 mock，使 `start_system_network_watcher` 返回错误。
2. target 基础间隔设置为 5 秒。
3. Wi-Fi 或有线初始正常。

## 测试场景

1. 系统监听器启动立即失败。
2. 无系统事件情况下断开网络。
3. 无系统事件情况下恢复网络。

## 执行步骤

1. 使用故障注入构建启动 watcher。
2. 确认初始快照正常。
3. 断开并恢复网络，观察定时更新。

## 通过要求

1. `startWatching()` 不因系统 watcher 初始化失败而失败。
2. run loop 继续运行。
3. 网络状态最终通过公网或 target 定时唤醒更新。
4. 不产生重复后台任务。
5. 日志应能诊断系统监听器已降级。

## 反例与失败判定

1. 系统监听失败导致插件完全无法启动，判定失败。
2. 降级后永远不再读取网络快照，判定失败。
3. 为重试监听器不断创建线程或句柄，判定失败。
