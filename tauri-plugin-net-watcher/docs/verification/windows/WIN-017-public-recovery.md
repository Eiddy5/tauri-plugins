# WIN-017 公网从不可用恢复

**基线状态：应通过**

## 测试用途

验证公网已经进入 unavailable 和退避后，网络恢复能够被及时发现，并立即恢复公网和 target 探测。

## 测试背景

失败退避不能导致恢复延迟过长。系统网络变化或下一次重试成功都应清除旧失败状态。

## 测试输入条件

1. 按 WIN-014 或 WIN-016 让公网进入 unavailable。
2. 至少一个 target 处于 suspended。
3. 准备恢复 WAN、解除防火墙或切换到健康网络。

## 测试场景

1. 恢复同一接口的 WAN，不产生物理网卡切换。
2. 从无公网 Wi-Fi 切换到正常有线。
3. 在 60 秒退避挡位期间触发系统网络变化。

## 执行步骤

1. 保存 unavailable 快照和当前退避级别。
2. 恢复公网并记录时间。
3. 观察网络事件和 target 事件。
4. 读取最终快照。

## 通过要求

1. 系统网络变化发生时立即重置公网挡位并探测，不等待旧 60 秒截止。
2. 任意一次主动探测成功立即返回 `available/verified=true`。
3. `consecutiveFailures` 清零。
4. suspended targets 立即恢复探测。
5. target 恢复数据属于新网络环境且质量窗口已重置。

## 反例与失败判定

1. 公网已恢复但必须等待完整退避周期，判定失败。
2. activeProbe 已成功但状态仍 unavailable，判定失败。
3. targets 未退出暂停，判定失败。

## 证据留存

1. 保存恢复前后快照。
2. 保存恢复操作和首个成功探测的时间差。
3. 保存网络与 target 事件顺序。
