# WIN-039 MAC 地址不采集

**基线状态：应通过**

## 测试用途

验证公开配置不能启用 MAC 地址采集，快照不会意外暴露设备硬件地址。

## 测试背景

MAC 地址不是核心需求，设计明确默认不采集，以降低隐私和设备标识风险。

## 测试输入条件

1. Windows 设备至少有一张具有物理 MAC 的网卡。
2. 使用默认配置启动插件。
3. 另准备包含未知字段或尝试启用 MAC 采集的错误配置。

## 测试场景

1. 默认 Wi-Fi 快照。
2. 默认有线快照。
3. 配置中添加 `includeMacAddress=true`。

## 执行步骤

1. 在默认配置下读取完整接口快照。
2. 搜索所有 `addresses.mac`。
3. 使用尝试启用采集的配置重新启动。

## 通过要求

1. 所有公开快照中的 `addresses.mac` 为 null。
2. 公开配置模型不暴露 includeMacAddress。
3. 未知字段因严格配置解析返回 `invalid_config`。
4. 事件 payload 中同样不包含真实 MAC 地址。

## 反例与失败判定

1. 任一快照或事件包含真实硬件地址，判定失败。
2. 前端可通过未声明配置开启采集，判定失败。
3. 日志打印完整 MAC 地址，判定失败。

## 证据留存

1. 保存脱敏后的快照字段检查结果。
2. 保存错误配置和返回错误。
3. 保存日志扫描结果。
