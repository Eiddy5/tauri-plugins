# WIN-005 Wi-Fi 主动断开

**基线状态：应通过**

## 测试用途

验证 watcher 运行期间断开唯一 Wi-Fi 后，设备、公网和 targets 能快速切换到正确状态并推送事件。

## 测试背景

系统网络事件必须优先于常规定时轮询触发刷新，且旧网络环境中的探测结果不能回写。

## 测试输入条件

1. 仅启用 Wi-Fi，公网和至少一个 target 初始为正常状态。
2. target 基础间隔设置为 10 秒或更长。
3. 页面记录网络事件和 target 事件到达时间。

## 测试场景

1. 通过 Windows 快捷设置点击“断开连接”。
2. 关闭 Wi-Fi 开关。
3. 开启飞行模式。

## 执行步骤

1. 等待插件显示公网和 targets 正常。
2. 记录断开操作时间并执行一种断开场景。
3. 等待网络事件和 target 暂停事件。
4. 保持断网 30 秒。

## 通过要求

1. 无需等待完整 `intervalMs` 即收到网络变化结果。
2. 最终 `overall=offline`、`network=disconnected`、`internet=unavailable`。
3. 主接口不再保持为可用 Wi-Fi。
4. targets 变为 `unknown/probe_suspended_network_unavailable`，不累计服务失败。
5. 断开前仍在执行的成功探测不得在断开后覆盖新状态。
6. 连续系统回调被合并，不产生事件风暴。

## 反例与失败判定

1. target 已暂停但公网仍长时间显示 `available`，判定失败。
2. 断开后出现旧 target `reachable` 事件，判定失败。
3. 同一次断开产生大量内容相同的网络事件，判定失败。

## 证据留存

1. 保存操作时间和所有事件时间戳。
2. 保存断开前后两份完整快照。
3. 记录状态完成收敛所用时间。
