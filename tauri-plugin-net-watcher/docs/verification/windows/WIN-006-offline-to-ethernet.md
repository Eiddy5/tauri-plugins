# WIN-006 断网状态插入网线

**基线状态：应通过**

## 测试用途

验证从完全断网状态插入可联网网线后，接口类型、公网和 targets 都立即重置并恢复。

## 测试背景

该场景用于防止主接口继续保留旧 Wi-Fi、target 已恢复但公网仍停留在不可用等跨层状态不一致。

## 测试输入条件

1. 初始关闭 Wi-Fi、拔出网线并断开 VPN。
2. 准备一条可通过 DHCP 获取地址并访问公网的网线。
3. 配置至少一个可达 target。
4. watcher 已运行且当前显示离线。

## 测试场景

1. 从完全断网直接插入网线。
2. 飞行模式关闭后立即插入网线。
3. 网线插入后 DHCP 地址延迟数秒获得。

## 执行步骤

1. 保存离线快照。
2. 记录时间并插入网线。
3. 等待 Windows 完成 DHCP。
4. 观察网络事件、target 事件和最终快照。

## 通过要求

1. 主接口最终为 `type=ethernet`，不能继续显示旧 Wi-Fi。
2. IPv4、网关和 DNS 更新为有线网络数据。
3. 公网在系统变化后主动重新探测，最终为 `available/verified=true`。
4. targets 在公网恢复后立即退出暂停并重新探测。
5. target 的旧质量窗口被清空，新样本属于有线网络环境。
6. 网络事件先于对应的新 target 探测结果到达。

## 反例与失败判定

1. target 已显示 reachable，但公网仍长期 unavailable，判定失败。
2. 网络类型仍为 Wi-Fi，判定失败。
3. 沿用断网前 target 统计而未重置，判定失败。

## 证据留存

1. 保存离线、DHCP 处理中和恢复后三份快照。
2. 保存事件顺序和时间戳。
3. 保存 `ipconfig /all` 对照数据。
