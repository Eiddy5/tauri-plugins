# WIN-010 Windows 睡眠与恢复

**基线状态：应通过**

## 测试用途

验证系统睡眠、待机或合盖后恢复时，watcher 能重新读取接口并主动验证公网与 targets。

## 测试背景

睡眠期间计时器暂停、套接字失效、DHCP 和 Wi-Fi 会重新协商。恢复后不能继续使用睡眠前的探测证据。

## 测试输入条件

1. 笔记本或支持睡眠的 Windows 测试机。
2. 初始通过 Wi-Fi 或有线联网，至少配置一个 target。
3. watcher 已运行并达到健康状态。

## 测试场景

1. 睡眠 30 秒后在同一网络恢复。
2. 睡眠期间关闭原路由器，恢复后连接另一个网络。
3. 合盖 5 分钟后恢复。

## 执行步骤

1. 保存睡眠前快照。
2. 让系统进入睡眠并等待指定时间。
3. 唤醒系统，等待网络恢复。
4. 观察第一批网络和 target 事件。

## 通过要求

1. 恢复后 watcher 仍在运行，无需再次 `startWatching()`。
2. 接口、地址、网关和 DNS 重新读取。
3. 网络发生变化时旧探测和质量窗口被废弃。
4. 公网和 targets 在恢复后重新探测并达到真实状态。
5. 不集中补发睡眠期间本应执行的定时探测。

## 反例与失败判定

1. 恢复后状态永久停留在睡眠前数据，判定失败。
2. watcher 任务退出或重复启动，判定失败。
3. 恢复瞬间大量补发事件导致前端卡顿，判定失败。

## 证据留存

1. 保存睡眠前后快照。
2. 保存事件时间戳并标记睡眠区间。
3. 记录 Windows 电源模式和网卡型号。
