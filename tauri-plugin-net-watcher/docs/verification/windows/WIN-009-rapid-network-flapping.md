# WIN-009 快速网络抖动与旧探测废弃

**基线状态：应通过**

## 测试用途

验证网络在短时间内连续断开、恢复和切换时，最终快照只反映最后一个稳定网络，旧探测结果全部失效。

## 测试背景

网卡切换会与 3 到 5 秒的公网/target 请求并发发生。如果旧请求完成后仍写入，就会产生状态倒退。

## 测试输入条件

1. 配置一个响应延迟 4 秒的测试 target。
2. 同时具备 Wi-Fi 和有线测试网络。
3. 页面记录每个事件的 `snapshotId`、时间和状态。

## 测试场景

1. target 探测开始后立即断开 Wi-Fi并插入网线。
2. 2 秒内连续执行 Wi-Fi 关闭、开启、再关闭。
3. 网线快速插拔三次，最后保持连接。

## 执行步骤

1. 等待慢 target 开始探测。
2. 执行一种快速抖动场景。
3. 等待最后网络稳定 15 秒。
4. 检查事件顺序和最终快照。

## 通过要求

1. 系统事件到达后，中断或丢弃旧网络中的在途探测。
2. 最终主接口、地址、网关和 DNS 对应最后稳定网络。
3. 最终状态稳定后，不再出现旧网络的成功或失败事件。
4. 每次真正的网络指纹变化都会清空 target 旧样本。
5. watcher 不 panic、不停止，也不出现事件风暴。

## 反例与失败判定

1. 最终网络稳定后出现旧 target 结果并覆盖状态，判定失败。
2. `snapshotId` 较新的事件携带旧接口数据，判定失败。
3. watcher 因通道关闭或任务异常退出，判定失败。

## 证据留存

1. 保存完整事件时间线。
2. 保存最终快照和系统网络状态。
3. 记录慢 target 服务端收到请求的时间。
