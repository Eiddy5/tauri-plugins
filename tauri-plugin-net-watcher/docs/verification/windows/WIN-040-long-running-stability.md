# WIN-040 Windows 长时间运行稳定性

**基线状态：应通过**

## 测试用途

验证 watcher 连续运行、周期探测和多次网络切换后，没有明显内存、线程、句柄、套接字或事件数量泄漏。

## 测试背景

网络插件通常贯穿应用整个生命周期。短期功能正确不能代替长期资源和调度稳定性。

## 测试输入条件

1. Windows 生产构建或接近生产的 release 构建。
2. 配置 4 个 targets，包含快、慢、成功和可控制失败服务。
3. 使用性能监视器记录进程内存、线程、句柄和网络连接。
4. 测试持续至少 8 小时，推荐 24 小时。

## 测试场景

1. 网络稳定运行 8 小时。
2. 每 30 分钟执行一次 Wi-Fi 断开恢复。
3. 每 2 小时执行一次 Wi-Fi/有线切换。
4. targets 周期性失败和恢复。
5. 系统睡眠恢复至少两次。

## 执行步骤

1. 启动应用并记录初始资源基线。
2. 按计划执行各场景并持续采样资源。
3. 保存所有结构化事件但避免页面无限渲染列表干扰插件指标。
4. 测试结束后停止 watcher并继续观察 10 分钟。

## 通过要求

1. 内存、线程、句柄和套接字数量无持续单调增长。
2. 同一时刻只有一个 watcher 和一个 Windows 系统监听注册。
3. 事件速率符合实际探测和语义变化，不出现失控增长。
4. 所有网络切换后最终状态正确，无永久卡死。
5. stop 后系统监听、任务和在途探测全部释放。
6. 全程无 panic、崩溃或未处理拒绝。

## 反例与失败判定

1. 每次网络切换永久增加线程或句柄，判定失败。
2. target 连接长期残留为 ESTABLISHED/SYN_SENT，判定失败。
3. stop 后仍持续请求或发送事件，判定失败。

## 证据留存

1. 保存资源趋势图和原始性能数据。
2. 保存场景操作日志与事件日志。
3. 保存崩溃、panic 和系统事件日志。
