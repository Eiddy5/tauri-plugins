# WIN-035 VPN 默认路由与主接口

**基线状态：预期失败/“主接口”领域定义待确认**

## 测试用途

验证 VPN 接管默认路由时，`primaryInterfaceId` 和网络指纹是否表达真实出口路径，而不是始终优先物理 Wi-Fi/有线。

## 测试背景

当前主接口通过接口类型优先级选择，物理接口优先于 VPN。若“主接口”定义为实际出口，该实现会产生错误数据。

## 测试输入条件

1. Windows 通过 Wi-Fi 或有线联网。
2. 准备全隧道 VPN 和分流 VPN。
3. 可使用 `route print` 或 `Get-NetRoute` 确认默认路由。

## 测试场景

1. 全隧道 VPN 接管 `0.0.0.0/0`。
2. 分流 VPN 不接管公网默认路由。
3. VPN 连接和断开但物理接口保持不变。
4. VPN 仅接管 IPv6 默认路由。

## 执行步骤

1. 保存 VPN 前快照和路由表。
2. 连接 VPN并等待路由稳定。
3. 保存 VPN 中快照和路由表。
4. 断开 VPN并再次观察。

## 通过要求

1. 先在领域文档中明确主接口是“物理接入介质”还是“实际默认出口”。
2. 若定义为实际出口，全隧道 VPN 应成为主接口并触发网络指纹重置。
3. 若定义为物理介质，应新增独立出口路由字段，避免 `primary` 名称误导。
4. VPN 连接变化必须让公网和 targets 在新路径上重新验证。

## 反例与失败判定

1. **当前已知反例：**物理接口类型优先级高于 VPN，可能与默认路由冲突。
2. VPN 路径改变但未重置探测窗口，判定失败。
3. 公网和 target 数据继续沿用 VPN 前样本，判定失败。

## 证据留存

1. 保存三阶段快照和路由表。
2. 保存 VPN 类型、客户端和版本。
3. 保存出口 IP 变化证据。
