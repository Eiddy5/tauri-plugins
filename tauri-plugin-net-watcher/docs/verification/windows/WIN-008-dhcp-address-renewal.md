# WIN-008 DHCP 地址、网关和 DNS 更新

**基线状态：应通过**

## 测试用途

验证接口保持连接但 DHCP 地址、默认网关或 DNS 发生变化时，网络指纹能够识别变化并重新验证。

## 测试背景

网络变化不只包含网卡上下线。租约续期、切换 VLAN 或路由器配置变化也会改变实际网络路径。

## 测试输入条件

1. Windows 设备连接到可控制 DHCP 的测试网络。
2. watcher 已运行且公网、targets 初始正常。
3. 新旧租约至少在 IPv4、网关、DNS 中有一个字段不同。

## 测试场景

1. 执行 `ipconfig /release` 后执行 `ipconfig /renew`。
2. 在同一网卡上切换到另一个 VLAN。
3. 修改 DHCP 下发的 DNS 后续租约。

## 执行步骤

1. 保存初始快照和 `ipconfig /all`。
2. 执行一种网络参数变化。
3. 等待 Windows 参数更新并观察插件事件。
4. 再次读取完整快照。

## 通过要求

1. 快照中的地址、网关和 DNS 更新为系统当前值。
2. 网络指纹变化触发公网立即重新验证。
3. target 质量窗口和调度被重置。
4. 不保留已释放的 IPv4 地址。
5. 最终公网和 targets 与新网络真实结果一致。

## 反例与失败判定

1. 仅接口状态变化被识别，而地址/DNS/网关变化未识别，判定失败。
2. 新旧 DNS 混合存在于同一接口，判定失败。
3. 直到常规 target 周期才刷新，且系统事件未提前唤醒，判定失败。

## 证据留存

1. 保存变化前后快照与 `ipconfig /all`。
2. 保存网络事件时间戳。
3. 记录 DHCP 操作时间。
