# WIN-026 target 自适应探测挡位

**基线状态：应通过；`intervalMs<3000` 的固定 3 秒契约参考反例说明**

## 测试用途

验证每个 target 根据健康、失败、恢复和质量不稳定状态独立调整下一次探测时间。

## 测试背景

动态调度用于兼顾故障发现速度与长期资源消耗，不能使用一个全局轮询周期控制所有 targets。

## 测试输入条件

1. 配置两个可独立控制的 targets。
2. `intervalMs=10000`、`timeoutMs=3000`。
3. 记录每次 target 事件到达时间和探测状态。

## 测试场景

1. target A 健康，target B 连续失败。
2. B 第一次和第二次失败。
3. B 第三次及更多失败。
4. B 首次恢复成功后再次成功。
5. A 持续高延迟但可达。

## 执行步骤

1. 启动 watcher并收集健康基线。
2. 按顺序控制 B 失败和恢复。
3. 独立制造 A 高延迟。
4. 计算各 target 相邻探测间隔。

## 通过要求

1. 健康 target 按基础间隔 ±10% 抖动运行。
2. 前两次失败约 3 秒确认。
3. 第三次失败后按 10、20、40、60 秒退避。
4. 首次稳定恢复后约 3 秒再次确认。
5. 高延迟或 unstable target 不高于 5 秒复查。
6. A 与 B 的挡位互不影响。

## 反例与失败判定

1. **契约反例：**当 `intervalMs=1000..2999` 时，当前实现使用 `min(3s, base)`，会小于文档固定 3 秒；应单独记录结果。
2. B 失败导致 A 进入退避，判定失败。
3. 持续失败仍按健康周期运行，判定失败。

## 证据留存

1. 保存每个 target 的探测时间线。
2. 保存服务端状态切换时间。
3. 保存计算后的实际间隔和容差。
