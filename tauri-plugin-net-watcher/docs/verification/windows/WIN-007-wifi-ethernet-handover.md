# WIN-007 Wi-Fi 与有线网络切换

**基线状态：应通过；VPN 路由不属于本用例**

## 测试用途

验证 watcher 运行期间在 Wi-Fi 和有线之间双向切换，主接口、网络指纹、公网与 target 数据不会交叉污染。

## 测试背景

Windows 切换物理接入介质时会产生多次接口回调，插件需要合并事件并只保留最终稳定网络的数据。

## 测试输入条件

1. Wi-Fi 和有线网络都可独立访问公网，且网段、网关或 DNS 至少一项不同。
2. 初始只连接 Wi-Fi。
3. 配置至少两个 targets。

## 测试场景

1. Wi-Fi 在线时插入网线，随后关闭 Wi-Fi。
2. 有线在线时开启 Wi-Fi，随后拔出网线。
3. 插拔网线与 Wi-Fi 开关在 1 秒内连续发生。

## 执行步骤

1. 记录初始 Wi-Fi 快照。
2. 执行一种切换场景。
3. 等待接口和公网稳定。
4. 反向切换并再次观察。

## 通过要求

1. 最终主接口类型与实际保留的物理连接一致。
2. 地址、网关和 DNS 来自最终网络，不能混用两个网络的数据。
3. 每次网络指纹变化都会重置公网和 target 调度。
4. 旧网络中的未完成探测不会覆盖新快照。
5. targets 在新网络上重新建立独立质量窗口。
6. 回调突发被 500ms 合并，不产生大量重复事件。

## 反例与失败判定

1. 主接口、类型、网关分别来自不同网络，判定失败。
2. 切换后仍显示上一个网络的 IP，判定失败。
3. 旧探测结果在切换完成后到达并改变状态，判定失败。

## 证据留存

1. 保存每次切换前后的快照。
2. 保存事件顺序、数量和时间戳。
3. 保存两个网络各自的 `ipconfig /all` 输出。
