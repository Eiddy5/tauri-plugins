# WIN-031 watcher 启停与重复调用

**基线状态：顺序调用应通过；并发 stop/start 当前存在已知反例**

## 测试用途

验证 watcher 顺序启动、停止、重复调用和并发启停时只有一个后台会话能够写入快照和发送事件。

## 测试背景

生命周期错误会产生重复探测、旧会话事件覆盖和资源泄漏，是生产稳定性的关键边界。

## 测试输入条件

1. watcher 初始停止。
2. 配置一个慢 target，便于观察任务重叠。
3. 前端可以并发发起 Tauri invoke。

## 测试场景

1. start → stop → start → stop 顺序执行。
2. 已启动时再次 start。
3. 未启动时 stop。
4. stop 尚未完成时并发 start。
5. 两个窗口同时调用 start。

## 执行步骤

1. 依次执行每个场景并记录返回值。
2. 对并发场景添加唯一会话 target 配置，识别旧新事件。
3. 停止后继续观察 10 秒。

## 通过要求

1. 正常顺序调用每次都能完成并重新启动。
2. 重复 start 返回 `already_watching`。
3. 未启动 stop 返回 `not_watching`。
4. 任意时刻最多一个 run loop 写快照和发送事件。
5. stop 完成后不再产生该会话事件，后台任务在 5 秒内退出或被中止。

## 反例与失败判定

1. **已知反例：**当前 stop 在等待旧任务退出前已移除共享任务；并发 start 可能启动第二个 run loop。该场景预期失败，修复前不得标记整项通过。
2. stop 后仍持续探测，判定失败。
3. 重复 start 静默创建第二个 watcher，判定失败。

## 证据留存

1. 保存每次 API 返回或错误 JSON。
2. 保存进程内事件时间线和 target 服务端并发日志。
3. 保存 stop 完成后的网络活动记录。
