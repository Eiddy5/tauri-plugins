# WIN-014 本地网络连通但无公网

**基线状态：应通过；失败阈值行为同时参考 WIN-016**

## 测试用途

验证设备已连接路由器但上游互联网断开时，本地接口仍为在线，而公网最终为不可用。

## 测试背景

“连接局域网”和“真实公网可用”必须分开，不能因为存在 IP、网关或 NCSI 缓存就错误显示公网可用。

## 测试输入条件

1. Windows 连接到可控制的 Wi-Fi 路由器或交换网络。
2. 保留 DHCP、局域网网关和 DNS配置，但断开路由器 WAN。
3. 配置至少一个公网 target。

## 测试场景

1. 断开路由器 WAN 但保留 LAN。
2. 防火墙阻断所有外网出口但允许本地网段。
3. 上游 DNS 和公网均不可用。

## 执行步骤

1. 先让插件达到公网 available。
2. 断开上游公网，保持本地接口不变。
3. 等待公网探测完成失败确认。
4. 读取最终快照并尝试访问局域网网关。

## 通过要求

1. `state.overall=online`、`state.network=connected` 保持不变。
2. 主接口、IP 和本地网关仍存在。
3. `state.internet` 最终收敛为 `unavailable`，`verified=false`。
4. `activeProbe.status=failed` 并提供结构化错误。
5. 公网不可用后 targets 进入暂停，而不是累计为各服务故障。

## 反例与失败判定

1. 本地接口连接导致公网持续显示 available，判定失败。
2. 直接把设备状态改为 offline，判定失败。
3. 每个公网 target 独立累计失败并被当成服务宕机，判定失败。

## 证据留存

1. 保存 WAN 断开前后快照。
2. 保存本地网关可达和公网不可达证据。
3. 保存公网探测错误码和时间线。
