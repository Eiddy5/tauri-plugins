# WIN-032 双栈 DNS 地址回退

**基线状态：预期失败/待修复**

## 测试用途

验证 target 域名解析出多个 IPv4/IPv6 地址时，一个候选地址失败不会导致整个 target 被误判为不可达。

## 测试背景

当前实现只连接 DNS 返回的第一个地址。双栈网络中第一个 IPv6 或 IPv4 路径失败、其他地址可用是常见情况。

## 测试输入条件

1. 准备同时具有 A 和 AAAA 记录的测试域名。
2. 让 DNS 返回的第一个地址不可连接，另一个地址可正常返回 HTTP 200。
3. 公网验证保持 available。
4. 记录系统 DNS 返回顺序。

## 测试场景

1. 第一个 IPv6 失败，后续 IPv4 成功。
2. 第一个 IPv4 失败，后续 IPv6 成功。
3. 同一地址族多个 A 或 AAAA 中第一个失败。

## 执行步骤

1. 使用 `Resolve-DnsName` 确认候选地址和顺序。
2. 使用防火墙只阻断第一个候选地址。
3. 启动 watcher并等待 target 探测。
4. 使用浏览器或支持地址回退的客户端验证服务可访问。

## 通过要求

1. 插件在总体 `timeoutMs` 内尝试后续候选地址。
2. 任一候选成功并返回合法 HTTP 状态时，target 为 reachable。
3. duration 和错误信息能够反映回退过程，不泄漏前一次失败为最终状态。
4. 所有候选都失败时才判定 unreachable。

## 反例与失败判定

1. **当前已知反例：**实现调用 DNS 结果 `.next()`，首地址失败即返回 `tcp_failed`。
2. 首地址失败而浏览器可访问时仍报告 unreachable，判定失败。
3. 每个候选分别使用完整 timeout 导致总时长无界，判定失败。

## 证据留存

1. 保存 DNS 结果顺序。
2. 保存防火墙规则、浏览器成功证据和 target JSON。
3. 记录各地址连接尝试时间。
