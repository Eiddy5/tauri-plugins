# WIN-016 公网连续失败确认与退避

**基线状态：严格契约应通过；当前实现对 `none/localAccess` 首次失败可能提前 unavailable，属于已知反例**

## 测试用途

验证公网主动探测从健康转为失败时，按快速确认和有界退避策略调度，并在连续第三次失败后收敛。

## 测试背景

单次丢包不应立即冻结所有 targets。稳定性策略要求短暂故障快速复核，持续故障降低探测频率。

## 测试输入条件

1. 初始公网为 available。
2. 使用防火墙或测试代理仅阻断 Microsoft Connect Test。
3. 精确记录 `checkedAt`、`consecutiveFailures`、status 和事件时间。
4. 本地接口始终保持连接。

## 测试场景

1. NCSI 仍为 `internetAccess`，主动探测连续失败。
2. NCSI 变为 `none/localAccess`，主动探测连续失败。
3. 探测超时而非立即拒绝。

## 执行步骤

1. 保存健康公网快照。
2. 开始阻断主动探测。
3. 观察前三次失败及后续至少四次重试。
4. 计算相邻 `checkedAt` 间隔。

## 通过要求

1. 前两次失败按约 3 秒复核，状态不应因单次瞬态故障直接稳定为 unavailable。
2. 第三次连续失败后 `status=unavailable`、`consecutiveFailures=3`。
3. reason 为 `internet_probe_failure_threshold_reached`。
4. 持续失败后的重试间隔依次约为 10、20、40、60 秒，并以 60 秒为上限。
5. 每次实际状态或失败计数变化产生的网络事件可与快照对应。

## 反例与失败判定

1. **已知反例：**系统提示为 `none/localAccess` 时，当前分类可能在第一次失败就返回 unavailable；应记录为失败，不能修改本用例通过标准。
2. 失败后仍每 3 秒无限探测，判定失败。
3. 退避超过 60 秒或失败计数回退，判定失败。

## 证据留存

1. 保存每次探测的时间、状态、原因和连续失败数。
2. 保存 NCSI 提示变化。
3. 保存防火墙规则及启停时间。
