# WIN-036 Windows 适配器 FFI 缓冲区安全

**基线状态：预期失败/静态缺陷待修复**

## 测试用途

验证传给 `GetAdaptersAddresses` 的缓冲区满足 `IP_ADAPTER_ADDRESSES_LH` 对齐、大小和生命周期要求。

## 测试背景

当前代码使用 `Vec<u8>` 后转换并解引用为 Windows 结构体。Rust 只保证字节对齐，存在未定义行为风险，普通功能测试不一定能复现。

## 测试输入条件

1. Windows x64 和 Windows ARM64 构建环境。
2. 启用严格静态检查、Miri 可覆盖部分或等价的 FFI 内存检查工具。
3. 准备拥有较多虚拟和物理适配器的测试机，使 API 触发缓冲区扩容。

## 测试场景

1. 初始 15000 字节缓冲区足够。
2. API 返回 `ERROR_BUFFER_OVERFLOW` 后重新分配。
3. x64 与 ARM64 不同对齐要求。
4. 高频重复读取适配器列表。

## 执行步骤

1. 静态检查分配类型、指针对齐和解引用位置。
2. 增加断言验证指针地址满足 `align_of::<IP_ADAPTER_ADDRESSES_LH>()`。
3. 在两种架构执行重复接口读取压力测试。
4. 运行可用的内存或未定义行为检测工具。

## 通过要求

1. 使用显式满足结构体对齐的分配方案。
2. 首次分配和 overflow 后重新分配都保持正确对齐。
3. 所有 Windows 结构体指针仅在缓冲区有效期内访问。
4. 压力测试无访问冲突、崩溃或内存错误。

## 反例与失败判定

1. **当前已知反例：**`Vec<u8>` 的契约不能保证目标结构体对齐，修复前本用例不得标为通过。
2. 仅因当前机器地址碰巧对齐就标记通过，属于无效验证。
3. overflow resize 后旧指针仍被使用，判定失败。
