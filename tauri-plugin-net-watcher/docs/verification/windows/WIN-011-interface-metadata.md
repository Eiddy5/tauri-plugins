# WIN-011 Windows 接口类型与元数据准确性

**基线状态：应通过**

## 测试用途

验证 Windows 系统适配器 API 采集的接口名称、类型、状态、地址、网关和 DNS 与系统真实数据一致。

## 测试背景

接口 `type` 曾出现 `unknown`，且断开的 Wi-Fi 可能残留地址。该用例专门验证基础采集层，不依赖公网和 target 结论。

## 测试输入条件

1. 测试机至少具有 Wi-Fi 和有线适配器。
2. 可选安装一个 VPN 或虚拟适配器用于扩展场景。
3. 准备 `Get-NetAdapter`、`Get-NetIPConfiguration` 和 `ipconfig /all` 作为对照。

## 测试场景

1. Wi-Fi 连接、有线断开。
2. 有线连接、Wi-Fi 断开。
3. Wi-Fi 与有线同时连接。
4. 存在 VPN、Hyper-V、WSL 或其他虚拟接口。

## 执行步骤

1. 在每个场景下保存系统命令输出。
2. 启动 watcher 并读取完整快照。
3. 逐接口核对字段。

## 通过要求

1. Wi-Fi 分类为 `wifi`，有线分类为 `ethernet`，VPN 分类为 `vpn`。
2. 接口 `status` 与 Windows `OperStatus` 一致。
3. 断开接口不能仅因残留地址被选为主接口。
4. IPv4、IPv6、网关和 DNS 与对应适配器一致。
5. 接口 ID 在多次读取中稳定且无重复。
6. 主接口只能有一个。

## 反例与失败判定

1. 已知 Wi-Fi/有线被分类为 `unknown`，判定失败。
2. 同一地址被错误合并到不同接口，判定失败。
3. 断开接口被标为 `up` 或主接口，判定失败。

## 证据留存

1. 保存完整接口 JSON。
2. 保存三个 Windows 对照命令输出。
3. 记录虚拟网卡软件和版本。
