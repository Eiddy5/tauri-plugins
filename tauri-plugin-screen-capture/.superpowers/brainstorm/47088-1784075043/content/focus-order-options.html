修复方向

点击一个方案。推荐 A：公开 API、无需额外授权、延迟稳定在 100ms 内。

A

轻量 Z-order 采样（推荐）

复用现有 100ms timer，只读取公开 CGWindow ID 顺序；发现目标与 panel 失序时才读取完整信息并修复。无需辅助功能或输入监控授权。

B

全局鼠标事件触发

监听 mouseDown，在下一轮 RunLoop 检查并修复。事件驱动开销低，但点击来源过滤和系统授权行为更容易受 macOS 版本影响。

C

Accessibility AXObserver

监听目标窗口聚焦/移动事件，时效最好；但必须引导用户到系统设置授予辅助功能权限，部署与审核成本最高。

A 的生产优势

  • 不依赖私有 CGS
  • 不监听用户输入
  • 沿用当前 timer 生命周期
  • 只有失序时走完整纠正路径

A 的边界

  • 最坏约 100ms 后恢复，而非同一点击帧
  • 需用真实窗口回归测试验证 Window ID 顺序
